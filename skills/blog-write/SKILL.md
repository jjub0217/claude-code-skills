# blog-write

## 개요

주간 학습 로그(`~/Desktop/weekly-learning-log.md`)에서 블로그 후보로 태그된 주제를 선택하여, 기술 블로그 초안을 작성한다.

## Execution Flow

```
┌─────────────────────────────────────────────────────┐
│  Step 1: 블로그 후보 목록 조회                          │
│  ─ weekly-learning-log.md 에서 "블로그 후보" 섹션 파싱   │
├─────────────────────────────────────────────────────┤
│  Step 2: 주제 선택                                    │
│  ─ 사용자에게 후보 목록 제시 → 주제 선택 or 직접 입력     │
├─────────────────────────────────────────────────────┤
│  Step 3: 블로그 초안 작성 에이전트 (병렬 실행)            │
│  ┌────────────────────┬────────────────────┐         │
│  │  context-gatherer  │  outline-writer    │         │
│  │  (관련 자료 수집)    │  (아웃라인 작성)    │         │
│  └────────────────────┴────────────────────┘         │
│                    ↓                                  │
│  ┌─────────────────────────────────────────┐         │
│  │  draft-writer (초안 작성)                 │         │
│  └─────────────────────────────────────────┘         │
├─────────────────────────────────────────────────────┤
│  Step 4: 초안 저장 + 사용자에게 보고                     │
└─────────────────────────────────────────────────────┘
```

## Step 1: 블로그 후보 목록 조회

### 실행

1. `~/Desktop/weekly-learning-log.md` 파일을 읽는다
2. `### 블로그 후보` 섹션을 모두 파싱하여 후보 목록을 추출한다
3. 사용자가 특정 주제를 직접 지정한 경우 → Step 2 건너뛰고 Step 3로 이동

## Step 2: 주제 선택

### 사용자에게 후보 목록 제시

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
블로그 후보 목록

1. [Config-driven UI] "Config-driven UI란? B2B 프론트엔드의 핵심 패턴"
2. [크로스플랫폼] "같은 서비스를 웹과 앱으로 만들기 - Expo 선택기"
...

어떤 주제로 블로그를 작성할까요? (번호 또는 직접 입력)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Step 3: 블로그 초안 작성

### Agent 1: context-gatherer (관련 자료 수집)

- **역할**: 선택된 주제와 관련된 세션 히스토리, 코드, 문서를 수집
- **수집 대상**:
  - 학습 로그에서 해당 주제 관련 Q&A, 인사이트, 의사결정
  - 관련 세션 히스토리 파일에서 상세 대화 내용
  - 프로젝트 코드에서 관련 구현 부분 (있는 경우)
- **출력**: 블로그 작성에 필요한 컨텍스트 요약

### Agent 2: outline-writer (아웃라인 작성)

- **역할**: 블로그 글의 구조를 설계
- **아웃라인 구성**:
  - 도입: 왜 이 주제를 다루는지 (문제 상황, 동기)
  - 본론: 핵심 내용 (개념 설명, 비교 분석, 구현 과정)
  - 결론: 배운 점, 다른 개발자에게 전하는 팁
- **출력**: 섹션별 제목 + 각 섹션에 들어갈 핵심 포인트

### Agent 3: draft-writer (초안 작성)

- **역할**: context-gatherer와 outline-writer의 결과를 결합하여 블로그 초안 작성
- **실행 조건**: Agent 1, 2가 완료된 후 실행
- **작성 톤**:
  - 1인칭 시점 ("저는", "제가")
  - 경험 기반 서술 (이론만 나열하지 않고 실제 경험 중심)
  - 쉬운 설명 (주니어 개발자도 이해할 수 있게)
  - 코드 예시가 있으면 포함
- **출력**: 완성된 블로그 초안 (마크다운)

## Step 4: 초안 저장 + 보고

### 저장 위치

`~/Desktop/blog-drafts/{날짜}-{주제슬러그}.md`

예: `~/Desktop/blog-drafts/2026-03-04-config-driven-ui.md`

### 블로그 초안 형식

```markdown
---
title: "(블로그 제목)"
date: 2026-03-04
tags: [React, Config-driven UI, B2B]
status: draft
---

# (블로그 제목)

(본문)
```

### 보고 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
블로그 초안 작성 완료

주제: (선택한 주제)
제목: "(블로그 제목)"
분량: 약 N자 (읽기 시간 약 N분)

파일: ~/Desktop/blog-drafts/{파일명}.md

다음 단계:
- 초안 검토 후 수정 요청 가능
- 완성되면 블로그 플랫폼에 게시
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 주의사항

- 블로그는 **사용자의 경험과 관점**으로 작성한다 (AI가 쓴 느낌 최소화)
- 학습 로그의 Q&A와 인사이트를 기반으로 하되, 블로그에 맞게 재구성한다
- 코드 예시는 실제 프로젝트 코드를 참고하되, 민감한 정보는 제거한다
- 초안이므로 사용자가 검토/수정할 수 있도록 명확한 구조로 작성한다
- 블로그 후보가 없는 경우, 학습 로그 전체를 스캔하여 블로그로 쓸만한 주제를 제안한다
