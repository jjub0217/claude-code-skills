# study-roadmap

## 설정

| 구분 | 항목 | 설명 | 기본값 |
|------|------|------|--------|
| 선택 | 로드맵 출력 경로 | 생성된 로드맵 파일이 저장될 디렉토리 경로 | `~/Desktop/` |

## 개요

회사 채용공고의 자격요건/우대사항과 사용자의 이력서를 비교하여, **갭 분석 → 맞춤 프로젝트 제안 → 공부 로드맵**을 자동 생성한다.

## Execution Flow

```
┌─────────────────────────────────────────────────────┐
│  Step 1: 입력 수집                                    │
│  ─ 채용공고 (텍스트/URL/노션) + 이력서 (노션/텍스트)     │
├─────────────────────────────────────────────────────┤
│  Step 2: 분석 에이전트 (병렬 실행)                       │
│  ┌────────────────────┬────────────────────┐         │
│  │  job-parser        │  resume-parser     │         │
│  │  (채용공고 파싱)     │  (이력서 파싱)      │         │
│  └────────────────────┴────────────────────┘         │
├─────────────────────────────────────────────────────┤
│  Step 3: 갭 분석                                      │
│  ─ 자격요건 vs 현재 수준 비교 → 갭 도출                  │
├─────────────────────────────────────────────────────┤
│  Step 4: 로드맵 생성 에이전트 (병렬 실행)                │
│  ┌────────────────────┬────────────────────┐         │
│  │  project-suggester │  roadmap-builder   │         │
│  │  (프로젝트 제안)     │  (로드맵 작성)      │         │
│  └────────────────────┴────────────────────┘         │
├─────────────────────────────────────────────────────┤
│  Step 5: 결과물 저장 + 사용자에게 보고                   │
└─────────────────────────────────────────────────────┘
```

## Step 1: 입력 수집

### 채용공고 입력 방식

다음 중 하나로 입력받는다:

1. **직접 텍스트**: 사용자가 채용공고 내용을 직접 붙여넣기
2. **URL**: 채용공고 URL → WebFetch로 내용 추출
3. **노션 페이지**: 노션 페이지 ID/URL → Notion MCP로 내용 추출
4. **로컬 파일**: 파일 경로 → Read로 내용 추출

### 이력서 입력 방식

다음 중 하나로 입력받는다:

1. **노션 페이지**: 노션 페이지 ID/URL → Notion MCP로 내용 추출 (기본값)
2. **직접 텍스트**: 사용자가 이력서 내용을 직접 붙여넣기
3. **로컬 파일**: 파일 경로 → Read로 내용 추출

### 사용자에게 입력 요청

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
공부 로드맵 생성기

다음 정보가 필요합니다:

1. 채용공고: (텍스트, URL, 노션 페이지, 파일 경로)
2. 이력서: (텍스트, 노션 페이지, 파일 경로)

이미 제공된 정보가 있다면 바로 분석을 시작합니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 입력 자동 감지

- 사용자가 스킬 호출 시 채용공고/이력서를 함께 제공한 경우 → 자동으로 Step 2 진행
- 이전 세션에서 이미 이력서를 노션으로 공유한 적이 있다면 → 해당 노션 페이지를 기본값으로 사용

## Step 2: 분석 에이전트 (병렬 실행)

2개의 에이전트를 **동시에** 실행한다.

### Agent 1: job-parser (채용공고 파싱)

- **역할**: 채용공고에서 구조화된 요구사항을 추출
- **추출 항목**:
  - 회사명, 팀명, 포지션
  - **자격요건** (필수): 각 항목을 개별적으로 분리
  - **우대사항** (선택): 각 항목을 개별적으로 분리
  - 기술 스택 키워드
  - 팀/직무 특성 (B2B, B2C, 모바일, 웹 등)
- **출력 형식**:
  ```json
  {
    "company": "회사명",
    "team": "팀명",
    "position": "포지션",
    "requirements": [
      { "id": 1, "text": "자격요건 내용", "keywords": ["React", "TypeScript"] }
    ],
    "preferred": [
      { "id": 1, "text": "우대사항 내용", "keywords": ["A/B Test"] }
    ],
    "tech_stack": ["React", "TypeScript", "Next.js"],
    "domain": "B2B"
  }
  ```

### Agent 2: resume-parser (이력서 파싱)

- **역할**: 이력서에서 기술 역량과 프로젝트 경험을 추출
- **추출 항목**:
  - 기술 스택 (사용 경험이 있는 기술)
  - 프로젝트 경험 (각 프로젝트의 기술/역할/성과)
  - 경력 사항
  - 강점 영역
- **출력 형식**:
  ```json
  {
    "tech_stack": ["React", "TypeScript", "Next.js"],
    "projects": [
      {
        "name": "프로젝트명",
        "tech": ["React", "WebSocket"],
        "role": "프론트엔드",
        "highlights": ["실시간 채팅 구현", "알림 시스템"]
      }
    ],
    "experience_years": 1,
    "strengths": ["실시간 통신", "React 컴포넌트 설계"]
  }
  ```

## Step 3: 갭 분석

Agent 1, 2의 결과를 결합하여 갭 분석을 수행한다.

### 분석 방법

각 자격요건/우대사항에 대해:

1. **현재 수준 평가**: 이력서의 프로젝트/경력에서 관련 경험이 있는지 확인
2. **갭 크기 판단**:
   - **충족**: 이력서에서 직접적인 경험이 확인됨
   - **부분 충족**: 관련 경험은 있으나 정확히 일치하지 않음
   - **미충족**: 관련 경험이 없거나 매우 부족
3. **우선순위 결정**: 자격요건의 갭이 우대사항보다 우선

### 출력 형식

```markdown
| # | 요구사항 | 현재 수준 | 갭 | 우선순위 |
|---|---------|----------|-----|---------|
| 1 | (요구사항) | (이력서 근거) | 충족/부분충족/미충족 | 높음/중간/낮음 |
```

## Step 4: 로드맵 생성 에이전트 (병렬 실행)

갭 분석 결과를 기반으로 2개의 에이전트를 **동시에** 실행한다.

### Agent 3: project-suggester (프로젝트 제안)

- **역할**: 갭을 채울 수 있는 구체적인 프로젝트를 제안
- **제안 기준**:
  - 기존 프로젝트를 확장하는 방향 우선 (새로 만드는 것보다 효율적)
  - 하나의 프로젝트로 여러 갭을 동시에 채울 수 있으면 우선
  - 포트폴리오/면접에서 어필할 수 있는 형태
- **각 프로젝트에 포함할 내용**:
  - 프로젝트명, 목표
  - 커버하는 자격요건/우대사항 (갭 분석 참조)
  - 기술 스택
  - 핵심 구현 범위
  - 면접 어필 포인트
- **출력 형식**: 마크다운 (프로젝트별 상세 설명)

### Agent 4: roadmap-builder (로드맵 작성)

- **역할**: 프로젝트를 기반으로 구체적인 공부 로드맵 작성
- **로드맵 유형** (2가지 제시):
  - **압축형**: AI를 적극 활용하여 빠르게 구현, 남는 시간에 코드 공부
  - **여유형**: 구현과 학습을 병행, 매주 AI가 만든 코드를 충분히 이해
- **각 일차/주차에 포함할 내용**:
  - 구현할 것
  - 학습 포인트 (면접에서 설명할 수 있어야 하는 것)
- **공통 섹션**:
  - 면접 대비 핵심 질문 예상 (5-10개)
  - 코드 공부 방법 (AI가 만든 코드를 이해하는 방법)
- **출력 형식**: 마크다운 (테이블 형식의 일정표)

## 주의사항

- 갭 분석은 이력서에 명시된 경험만 기준으로 한다 (추측하지 않음)
- 프로젝트 제안 시 기존 프로젝트 확장을 우선한다 (신규 프로젝트는 최후 수단)
- 로드맵은 현실적인 일정으로 작성한다 (AI 활용 시 단축 가능한 부분 명시)
- 면접 질문은 해당 회사/팀의 도메인에 맞게 작성한다
- 멘토 강조사항 반영: "AI가 만든 코드에 대해서 공부해야 한다. AI의 도움 없이도 설명할 수 있어야 한다."
