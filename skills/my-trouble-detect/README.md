# my-trouble-detect

에러/삽질 발생 시 컨텍스트를 자동 수집하고 분석하는 스킬.

## 기능
- 3개 병렬 에이전트: 에러 로그 수집, git 변경 이력 수집, 환경 정보 수집
- 패턴 분석 에이전트로 원인 추론 및 해결 방향 제안
- 과거 트러블 기록과 비교하여 반복 패턴 감지
- 트러블슈팅 리포트 자동 저장 (나중에 패턴 분석에 활용)

## 설정
| 구분 | 항목 | 기본값 |
|------|------|--------|
| 선택 | 트러블 리포트 경로 | `docs/troubles/` |

## 사용법
```
/my-trouble-detect
```

## 예시
빌드 에러가 발생했을 때 "에러 감지해줘"라고 요청하면, 에러 로그/변경 이력/환경 정보를 수집하고 원인을 분석하여 해결 방향을 제안합니다.
