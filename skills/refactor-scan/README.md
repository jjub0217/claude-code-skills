# refactor-scan

Best Practice 규칙 기반으로 코드를 스캔하고 리팩토링을 자동화하는 스킬.

## 기능
- Best Practice 규칙 기반 코드 자동 스캔 (병렬 패턴 탐색)
- 위반 사항 우선순위별 정리 및 리팩토링
- GitHub 이슈 → 브랜치 → 커밋 → PR 자동 생성
- Notion에 리팩토링 근거 문서 생성 (선택)

## 설정
| 구분 | 항목 | Notion 필요 |
|------|------|-------------|
| 필수 | 대상 프레임워크, 규칙 파일 경로, 스캔 대상 경로 | 불필요 |
| 필수 | GitHub 레포, base 브랜치 | 불필요 |
| 선택 | DAILY SCRUM DB, 사용자 ID/이름 | 필요 |
| 선택 | 블로그 초안 경로, 제외 경로 | 불필요 |

Notion 미설정 시 근거 문서 생성 단계를 건너뜁니다.

## 사용법
```
/refactor-scan
```

## 예시
"리팩토링 스캔해줘"라고 요청하면, 설정된 규칙으로 코드를 스캔하고 위반 사항을 보여준 뒤 확인하면 리팩토링 + PR 생성까지 자동 처리합니다.
