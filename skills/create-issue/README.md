# create-issue

GitHub 이슈 생성 + 브랜치 생성을 자동화하는 스킬.

## 기능
- 작업 내용 기반으로 이슈 타입 자동 판단 (feature/fix/refactor/docs)
- 이슈 템플릿에 맞춘 이슈 본문 자동 작성
- base 브랜치에서 네이밍 컨벤션에 맞는 브랜치 자동 생성

## 설정
| 구분 | 항목 | 기본값 |
|------|------|--------|
| 필수 | GitHub 레포 | - |
| 선택 | base 브랜치 | `develop` |
| 선택 | 브랜치 네이밍 형식 | `종류/이슈번호--이름` |
| 선택 | 라벨 매핑 | 프로젝트에 맞게 설정 |
| 선택 | 이슈 템플릿 경로 | `.github/ISSUE_TEMPLATE/` |

## 사용법
```
/create-issue
```

## 예시
"로그인 UI 작업할거야"라고 요청하면, feature 이슈를 생성하고 `feature/123--login-ui` 브랜치를 자동으로 만들어줍니다.
