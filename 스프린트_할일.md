# Claude Code Skills — 범용화 스프린트

## 목표
하드코딩된 프로젝트 정보를 설정으로 분리하고, Notion 없이도 동작하도록 로컬 마크다운 fallback을 추가한다.

---

## Sprint 1: 설정 구조 + bug-fix 범용화 (Day 1)

### 목표
공통 설정 구조를 정하고, bug-fix 스킬에 적용하여 패턴을 확립한다.

### 할일
- [ ] 1-1. 설정 구조 설계 — 각 스킬 SKILL.md 상단에 `## 설정` 섹션 추가, 필수/선택 구분
- [ ] 1-2. bug-fix 범용화 — Notion DB ID를 설정 섹션으로 이동
- [ ] 1-3. bug-fix Notion fallback — Notion 미설정 시 `qa/` 폴더에 마크다운 QA 티켓 생성
- [ ] 1-4. bug-fix GitHub 레포명 설정화 — 하드코딩된 레포명을 설정으로 분리
- [ ] 1-5. 커밋 + 푸시

### 완료 기준
- Notion 설정 있을 때: 기존과 동일하게 동작
- Notion 설정 없을 때: `qa/` 폴더에 마크다운 생성 + GitHub 이슈 + worktree 수정 + PR

---

## Sprint 2: daily-scrum + context-sync 범용화 (Day 2)

### 목표
daily-scrum과 context-sync에 로컬 fallback을 추가한다.

### 할일
- [ ] 2-1. daily-scrum 범용화 — Notion DB ID를 설정으로 이동
- [ ] 2-2. daily-scrum Notion fallback — Notion 미설정 시 `scrum/` 폴더에 마크다운 생성
- [ ] 2-3. context-sync 범용화 — Notion DB ID, GitHub 레포명을 설정으로 이동
- [ ] 2-4. context-sync Notion fallback — Notion 미설정 시 GitHub 수집 + 싱크 문서만 생성
- [ ] 2-5. 커밋 + 푸시

### 완료 기준
- Notion 없어도 daily-scrum, context-sync가 로컬 문서로 동작

---

## Sprint 3: create-issue + commit-push 범용화 (Day 3)

### 목표
create-issue와 commit-push에서 프로젝트 종속 설정을 분리한다.

### 할일
- [ ] 3-1. create-issue 범용화 — 브랜치 네이밍 컨벤션, 라벨 매핑을 설정으로 이동
- [ ] 3-2. create-issue 이슈 템플릿 설정화 — 템플릿 경로를 설정으로 분리
- [ ] 3-3. commit-push 범용화 — PR 템플릿, base 브랜치를 설정으로 이동
- [ ] 3-4. commit-push PR 형식 설정화 — chore/fix/feat별 PR body 형식을 설정으로 분리
- [ ] 3-5. 커밋 + 푸시

### 완료 기준
- 다른 프로젝트에서 설정만 바꾸면 create-issue, commit-push 사용 가능

---

## Sprint 4: 학습/유틸리티 스킬 범용화 (Day 4)

### 목표
learn-log, blog-write, study-roadmap, my-session-wrap, my-trouble-detect의 하드코딩 제거.

### 할일
- [ ] 4-1. learn-log 범용화 — 히스토리 파일 경로, 학습 로그 파일 경로를 설정으로 이동
- [ ] 4-2. blog-write 범용화 — 학습 로그 파일 경로를 설정으로 이동
- [ ] 4-3. study-roadmap 범용화 — 프로젝트 종속 정보 제거 (이미 범용적이면 확인만)
- [ ] 4-4. my-session-wrap, my-trouble-detect 범용화 — 프로젝트 종속 정보 제거
- [ ] 4-5. 커밋 + 푸시

### 완료 기준
- 5개 스킬 모두 설정만 바꾸면 어떤 프로젝트에서든 사용 가능

## Sprint 5: refactor-scan + README 업데이트 + 최종 정리 (Day 5)

### 목표
마지막 스킬 범용화 + 문서 정리 + 최종 점검.

### 할일
- [ ] 5-1. refactor-scan 범용화 — 프로젝트 종속 규칙이 있다면 설정으로 분리
- [ ] 5-2. README 업데이트 — 각 스킬별 설정 가이드 보강, 사용 예시 추가
- [ ] 5-3. 각 스킬 폴더에 개별 README.md 추가 — 스킬별 상세 설명 + 설정 방법
- [ ] 5-4. 전체 스킬 최종 점검 — 하드코딩 잔존 여부 확인
- [ ] 5-5. 최종 커밋 + 푸시

### 완료 기준
- 11개 스킬 모두 범용화 완료
- README에 설치/설정/사용 가이드 완비
- 하드코딩된 프로젝트 정보 0건
